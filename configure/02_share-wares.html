Share WARES on AAltSys Server¶

This basic guide describes moving WARES from a single user computer or other shared drive onto an AAltSys Server. This guide is appropriate when the WARES copy is stored within an ATLAS folder. When finished, WARES will be accessed from a server share mounted as drive W:\ at each workstation.

Note

This guide assumes that WARES patches and support files have been added to the Shared share or drive S:\. Make sure these files are present beforehand.

Prep WARES on the old server¶

Warning

WARES must be prepped on its original server if it is currently using a proprietary network driver.

  1. From a workstation, log into WARES using the supervisor level password.
  2. Press F2 to see all menus, choose STARTUP.
  3. Select Application ‣ Open, and enter the account SYSPROG, password _____________.
  4. In the Commands window, enter RUNMENU LANPACK.
  5. Choose Lanpack ‣ Network Type from the menu.
  6. Press F2 for options, and switch to Microsoft Lan Manager.
  7. Press F9 to save the setting, (A message will display saying revboot is rebuilt.)
  8. Choose Exit to exit the application.

Then copy the entire ATLAS directory from the old server to a flash drive or to a workstation C:\ drive. Call if you have questions about what to copy.

Install an AAltSys Server¶

Install an AAltsys Server, adding the following instructions specific to the WARES program.

Configure the AAltSys Server¶

  1. In Users and Computers, define groups WARES and FSUSERS. Define all system users and add them to groups as appropriate.
  2. In File Sharing, define the wares share as a directory wares under Zentyal, checking Apply ACLs Recursively.
  3. Set “administrator” Access Control permissions on this share for group “WARES”.
  4. At a workstation, test to see that this share is visible, and map this share to drive letter W:\.
  5. Define the shared share as a directory shared under Zentyal, and checking Apply ACLs Recursively.
  6. Set “read/write” Access Control permissions on this share for group “FSUSERS”.
  7. At a workstation, see that the share is visible, and map the share to drive letter S:\.

Setup WARES on the Server¶

Copy WARES to the Server¶

  1. At a workstation, open the saved ATLAS folder. If there is an AREV folder within ATLAS, open that folder.
  2. Press <Ctrl-A> <Ctrl-C> to select all files within the open folder.
  3. Open the drive W:\, and click in the window.
  4. Press Ctrl-V to paste the contents of the folder into drive W:\.
  5. Create folders W:\exchange and W:\export, if missing.
  6. Copy the contents of S:\arev to W\ (usually folder patches).
  7. If ATLAS contained an AREV folder, copy any other folders in ATLAS but not included in AREV, such as CONNECT, to W:\.

Configure WARES on the Server¶

  1. Copy W:WARES.pif to a workstation desktop.
  2. Double-click the WARES.pif icon, and log into WARES at the supervisor level.
  3. Select Tools ‣ Utilities ‣ Bundle from the menu.
  4. Press Shift-F1 to display the SetVolume window.
  5. Enter volume name PATCHES.
  6. If the data location is ..PATCHES, change it to .PATCHES (that is, remove one period).
  7. Press F9 to save changes and Esc to return to Bundle Records.
  8. Press Enter to accept the default volume name, PATCHES.
  9. At Filename, press F2 and select OTHER.
  10. Press Alt-T to move to Actions, press F2, and choose RESTORE.
  11. Press F9 to restore the WARES update.

Note

Install other PATCHES update bundles using the same procedure, if desired.
