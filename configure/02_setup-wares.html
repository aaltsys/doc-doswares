Share WARES on AAltSys Server¶

This basic guide describes moving WARES from a single user computer or other shared drive onto an AAltSys Server. This guide is appropriate when the WARES copy is stored within an ATLAS folder. When finished, WARES will be accessed from a server share mounted as drive W:\ at each workstation.

Note

This guide assumes that WARES patches and support files have been added to the Shared share or drive S:\. Make sure these files are present beforehand.

Prep WARES on the old server¶

Warning

WARES must be prepped on its original server if it is currently using a proprietary network driver.

  • From a workstation, log into WARES using the supervisor level password.
  • Press F2 to see all menus, choose STARTUP.
  • Select Application ‣ Open, and enter the account SYSPROG, password _____________.
  • In the Commands window, enter RUNMENU LANPACK.
  • Choose Lanpack ‣ Network Type from the menu.
  • Press F2 for options, and switch to Microsoft Lan Manager.
  • Press F9 to save the setting, (A message will display saying revboot is rebuilt.)
  • Choose Exit to exit the application.

Then copy the entire ATLAS directory from the old server to a flash drive or to a workstation C:\ drive. Call if you have questions about what to copy.

Install an AAltSys Server¶

Install an AAltsys Server, adding the following instructions specific to the WARES program.

Configure the AAltSys Server¶

  • In File Sharing, define the wares share as a directory arev under Zentyal.
  • Set “read/write” Access Control permissions on this share for group “Users” (or group “fsusers”).
  • At a workstation, test to see that this share is visible.
  • Map this share to drive letter W:.
  • Define the shared share as a directory public under Zentyal.
  • Set “read/write” Access Control permissions on this share for group “fsusers” (or group “Users”).
  • See that the share is visible, and map the share to drive letter S:\.

Setup WARES on the Server¶

Copy WARES to the Server¶

  • At a workstation, open the saved ATLAS folder. If there is an AREV folder within ATLAS, open that folder.
  • Press <Ctrl-A> <Ctrl-C> to select all files within the open folder.
  • Open the drive W:\, and click in the window.
  • Press Ctrl-V to paste the contents of the folder into drive W:\.
  • Create folders W:\exchange and W:\export, if missing.
  • Copy the contents of S:\arev to W\ (usually folder patches).
  • If ATLAS contained an AREV folder, copy any other folders in ATLAS but not included in AREV, such as CONNECT, to W:\.

Configure WARES on the Server¶

  • Copy W:WARES.pif to a workstation desktop.
  • Double-click the WARES.pif icon, and log into WARES at the supervisor level.
  • Select Tools ‣ Utilities ‣ Bundle from the menu.
  • Press Shift-F1 to display the SetVolume window.
  • Enter volume name PATCHES.
  • If the data location is ..PATCHES, change it to .PATCHES (that is, remove one period).
  • Press F9 to save changes and Esc to return to Bundle Records.
  • Press Enter to accept the default volume name, PATCHES.
  • At Filename, press F2 and select OTHER.
  • Press Alt-T to move to Actions, press F2, and choose RESTORE.
  • Press F9 to restore the WARES update.

Note

Install additional PATCHES update bundles using the same procedure, if desired.

Configure Workstations¶

Prepare each workstation to run WARES using workstation management guidelines. In addition, each workstation will require batch files and pif files as described at DOS Programs and NET USE and EMS Memory Under MS Windows.
